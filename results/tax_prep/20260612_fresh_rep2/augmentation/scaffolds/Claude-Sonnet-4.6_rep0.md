**Assistant Guidance: Three-Phase Workflow**

**Phase 1: Requirements Check**

Before any analysis begins, catalog all inputs and flag gaps:
- List every client-provided data point and every pre-filled return entry separately.
- Identify explicitly missing information (e.g., business expenses, any additional income sources, deductible items not yet provided).
- Note which items are potentially incorrect or inconsistent between what the client provided and what appears on the pre-filled returns.
- Flag any eligibility questions that require rule-based analysis before proceeding (filing status, dependent qualification, credit eligibility).
- Document what assumptions will be required where data is absent, so those assumptions are transparent throughout.

**Phase 2: Plan & Structure**

Organize the review into logical, sequential workstreams:
- Determine the correct order of operations (income → adjustments → deductions → credits → taxes → withholding/payments).
- Group issues by form or schedule affected, so corrections can be traced to specific lines.
- Identify which issues are interdependent (e.g., correcting income affects AGI, which cascades into deduction thresholds, credits, and state calculations).
- Plan how to handle each missing data point: state the gap, define the assumption, and outline the calculation framework that would apply once data is provided.

**Phase 3: Draft → Self-Review → Finalize**

- Draft the analysis issue-by-issue, following the structure from Phase 2.
- Self-review by tracing each correction forward through the return to confirm downstream effects are addressed consistently.
- Verify that every discrepancy identified in Phase 1 is either resolved or explicitly flagged as unresolvable without additional information.
- Confirm the response distinguishes clearly between definitive findings and estimate-based frameworks.
- Finalize with a summary of required corrections, missing information, and next steps.
