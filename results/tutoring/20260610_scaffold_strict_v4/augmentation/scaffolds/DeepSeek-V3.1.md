**Assistant Guidance: Three-Phase Workflow**

1.  **Requirements Check**
    Systematically parse the task prompt to identify all mandatory components (e.g., learning objective, explanation, analogy, examples, misconceptions, activity, checks, wrap-up). Verify the target audience (age, grade) and all constraints (e.g., age-appropriate language, avoidance of abstract notation, need for visual support). Confirm that the output must be classroom-ready, meaning it is a complete, self-contained segment for immediate use by an educator.

2.  **Plan & Structure**
    Outline a logical pedagogical flow that sequences the required components into a cohesive lesson arc, from introduction to conclusion. For each component, plan its core purpose and how it will be visually supported. Brainstorm simple analogies relatable to 8-year-olds and common errors for the specified topic. Design an interactive activity that is hands-on and minimizes passive listening. Structure checks for understanding as low-stakes, formative questions.

3.  **Draft -> Self-Review -> Finalize**
    Compose the initial draft using the planned structure, strictly adhering to the language and visual support constraints. Perform a self-review against the original task requirements, checking for missing elements, appropriateness for the age group, and clarity. Ensure all explanations are simple and all activities are feasible in a classroom setting. Refine the language for engagement and clarity, then format the final output for readability.
